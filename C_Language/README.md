## [W2] C언어 기초 & 컴파일

**[배운 내용]**
* **소프트웨어 개발 단계:** 요구사항 분석 -> 설계 -> 구현 -> 테스팅
* **C 프로그램 구조:** `#include <stdio.h>`, `main` 함수, 주석(`//`, `/* */`)
* **컴파일 & 링킹:** 소스(.c) -> 오브젝트(.obj) -> 실행(.exe)
* **입출력:** `printf`(출력), `scanf`(입력)

**[코드/개념 메모]**
* **기본 틀 (무조건 암기):**
    ```c
    #include <stdio.h> // 이거 필수

    int main(void) {
        printf("Hello World!\n"); 
        return 0;
    }
    ```
* **scanf 사용법:**
    * 변수 앞에 `&` 필수. (`scanf("%d", &num);`)
    * VS에서는 `scanf_s` 쓰라고 에러 나는데, `#define _CRT_SECURE_NO_WARNINGS` 맨 윗줄에 쓰면 해결됨.

**[주의/느낀점]**
* 세미콜론(`;`) 아주 중요!!
* `scanf`에 `&` 빼먹어서 프로그램 오류 남. 주의할 것.

## [W4] 연산자

**[배운 내용]**
* **산술:** `+`, `-`, `*`, `/`, `%`(나머지)
* **증감:** `++`, `--` (위치에 따라 값 달라짐)
* **관계/논리:** `==`, `!=`, `&&`(AND), `||`(OR)
* **형 변환:** `(int)3.14` 처럼 강제로 바꾸는 것.

**[코드/개념 메모]**
* **나눗셈 `/` 주의:** * `5 / 2` = `2` (정수끼리 나누면 소수점 버림)
    * `5.0 / 2` = `2.5` (실수 포함돼야 실수 나옴)
* **증감 연산자:** `a = ++x;` (증가 후 대입) vs `a = x++;` (대입 후 증가).
* **실수 비교:** `0.1 + 0.2 == 0.3` 은 거짓임. 오차 있어서 `==` 쓰면 안 됨.

**[주의/느낀점]**
* 수학처럼 `10 < x < 20` 쓰면 안되고, `10 < x && x < 20` 이렇게 써야 함.
* `%` 연산자는 정수만 가능능. 실수 넣으면 에러.

## [W5] 제어문 (if, switch)

**[배운 내용]**
* **구조:** 순차, 선택(if, switch), 반복
* **if문:** 조건에 따라 실행 (`if`, `else if`, `else`)
* **switch문:** 값에 따라 점프 (`case`, `break`, `default`)

**[코드/개념 메모]**
* **if문 실수 주의:**
    * `if (x = 0)` X -> 대입이 돼버림.
    * `if (x == 0)` O -> 비교 연산자 확인 필수.
* **switch문 주의:**
    * `case` 뒤에는 정수나 문자만 가능. (실수, 문자열 불가)
    * `break` 안 쓰면 밑에 있는 코드까지 줄줄이 실행됨 (Fall-through).

**[주의/느낀점]**
* `switch`문 쓸 때 `break` 빼먹지 말기.
* `if`문 안에 `if` 또 쓸 때 중괄호 `{}` 위치 잘 보기기.

## [W6] 반복문 (while, for)

**[배운 내용]**
* **while문:** 조건이 참인 동안 계속 반복.
* **for문:** 초기화, 조건, 증감이 한 줄에 있음. (횟수 정해진 경우 유리)
* **do-while문:** 일단 한 번 실행하고 조건 검사.
* **제어:** `break`(탈출), `continue`(건너뛰기)

**[코드/개념 메모]**
* **for문 기본:** `for (int i = 0; i < 10; i++)` -> 0부터 9까지 10번.
* **무한 루프:** `while(1)` -> 안 멈춤. `Ctrl + C`로 강제 종료.
* **while vs do-while:** `do-while`은 조건 안 맞아도 무조건 1번은 실행됨.

**[주의/느낀점]**
* `i`를 0부터 시작할지 1부터 시작할지 잘 생각해야 할 듯. (보통 0부터 하는 것 같음)
* 반복문 안에 `scanf` 넣었다가 종료 조건 안 만들어서 못 빠져나옴.

## [W7] 입출력 스트림 (I/O Stream)

**[배운 내용]**
* **스트림(Stream):** 데이터가 이동하는 통로. (키보드 -> 프로그램 -> 모니터/파일)
* **표준 입출력:** `stdin`(키보드), `stdout`(모니터). `getchar()`, `putchar()`.
* **파일 입출력:** 데이터를 텍스트 파일(.txt)에 저장하거나 불러오기.
* **핵심 함수:** `fopen`(열기), `fclose`(닫기), `fprintf`(파일에 쓰기), `fscanf`(파일에서 읽기).

**[코드/개념 메모]**
* **파일 열기 모드:**
    * `"r"` (Read): 읽기 전용. 파일 없으면 에러 남.
    * `"w"` (Write): 쓰기 전용. 파일 있으면 내용 싹 지우고 새로 만듦.(조심!)
    * `"a"` (Append): 이어 쓰기. 파일 끝에 내용 추가됨.
* **기본 패턴:**
    ```c
    FILE *fp = fopen("data.txt", "w"); // 파일 열기
    if (fp == NULL) {
        printf("파일 열기 실패!\n");
        return 1; // 예외 처리 필수
    }
    fprintf(fp, "Hello File!\n"); // 파일에 쓰기
    fclose(fp); // 파일 닫기 (안 하면 저장 안 됨)
    ```
* **EOF (End Of File):** 파일의 끝을 알리는 표시. `while(fscanf(...) != EOF)` 이렇게 많이 씀.

**[주의/느낀점]**
* `"w"` 모드로 열면 기존 내용 다 날아가기 때문에 백업 필수.
* `fclose(fp)` 까먹지 않기.

## [W9] 함수 (Definition & Calling)

**[배운 내용]**
* **함수 쓰는 이유:** 재사용성, 기능별로 쪼개려고 (모듈화).
* **구조:** 반환형(Return Type) + 함수명 + 매개변수(Parameter) + 몸체(Body).
* **함수 선언 (Prototype):** `main` 함수보다 밑에 만들 거면 위에서 미리 "나 이런 함수 있다"고 알려줘야 함.
* **라이브러리:** `#include <math.h>` 쓰면 sqrt(루트), pow(제곱), fabs(절대값) 사용 가능.

**[코드/개념 메모]**
* **용어 구분:**
    * **Parameter (매개변수):** 함수 정의할 때 괄호 안에 있는 변수 (`int x`).
    * **Argument (인자):** 함수 부를 때 넘겨주는 실제 값 (`3`, `num`).
* **반환형 주의:**
    * 반환값 없으면 `void` 써야 함.
    * `return` 만나면 함수 즉시 종료되고 값 반환.

**[주의/느낀점]**
* `void` 함수에서 `return 0;` 쓰면 안되고, 그냥 `return;`만 쓰거나 안 써야 함.

## [W10] 변수 유효범위 & 재귀함수

**[배운 내용]**
* **지역변수 (Local):** 중괄호 `{}` 안에서만 생존. 함수 끝나면 메모리에서 삭제됨.
* **전역변수 (Global):** 함수 바깥에 선언. 프로그램 끝날 때까지 살아있음. 어디서든 접근 가능.
* **정적변수 (Static):** `static int a = 0;` 처럼 씀. 지역변수인데 함수 끝나도 값 안 사라지고 유지됨.
* **재귀함수 (Recursion):** 자기가 자기 자신을 부르는 함수. (팩토리얼, 하노이탑 등)

**[코드/개념 메모]**
* **전역변수 특징:** 초기화 안 하면 자동으로 0 들어감. (지역변수는 쓰레기값 들어감).
* **재귀함수 필수 조건:**
    * **Base Case (탈출 조건):** 없으면 무한루프 돌다가 스택 오버플로우 남.
    * 예시: `if (n == 0) return;` 같은 거 필요.

**[주의/느낀점]**
* 전역변수 편하다고 막 쓰면 나중에 어디서 값이 바뀌었는지 모름. 최대한 사용x
* 재귀함수(하노이탑) 로직 어려워서 그냥 반복문 쓰는 게 편할 수도...

## [W11] 배열 (Array)

**[배운 내용]**
* **배열:** 변수 여러 개 만들지 않고 한번에 만드는 것. (아파트 우편함 같은 느낌)
* **인덱스(Index):** 방 번호. **무조건 0부터 시작.** (1부터 시작 아님!!!)
* **선언:** `int score[5];` -> 정수 방 5개 만듦 (0, 1, 2, 3, 4번 방).
* **초기화:** `int arr[3] = {10, 20, 30};` (중괄호 사용).

**[코드/개념 메모]**
* **인덱스 실수 (Out of Bounds):**
    * `int a[5];` 만들고 `a[5] = 10;` 하면 망함. (0~4까지만 있음)
    * 실행하면 에러 안 뜨고 그냥 이상한 값 나오거나 프로그램 뻗음. 젤 위험함.
* **2차원 배열:** `int arr[2][3];` -> 2행 3열짜리 표. (엑셀 생각하면 됨).
    * `arr[0][0]`, `arr[0][1]` ... 이런 식.

**[주의/느낀점]**
* `for`문 돌릴 때 `i <= 5`라고 썼다가 범위 넘어감. `i < 5`로 써야 함.
* 배열 크기 변수로 못 넣음 (`int size = 10; int arr[size];` -> 옛날 C에선 안 됨). 그냥 숫자 넣는 게 맘 편함.
* 
* 2차원 배열은 어려워서 많은 연습이 필요...

## [W12] 문자와 문자열 (String)

**[배운 내용]**
* **문자(char):** 작은따옴표 `'A'`, 1글자.
* **문자열(String):** 큰따옴표 `"Hello"`, 문자들의 배열.
* **널 문자(\0):** 문자열의 끝을 알리는 투명인간 같은 존재. (필수!)
* **함수:** `strlen`(길이), `strcpy`(복사), `strcmp`(비교), `strcat`(이어붙이기). -> `#include <string.h>` 필요.

**[코드/개념 메모]**
* **널 문자(Null Character):**
    * "Hi"는 2글자 같지만 실제론 `['H', 'i', '\0']` 3칸을 차지.
    * 배열 크기 잡을 때 무조건 **글자수 + 1** 해줘야 함.
* **문자열 비교:**
    * `if (str == "Hello")` -> 절대 안 됨. (주소 비교하는 거임)
    * `if (strcmp(str, "Hello") == 0)` -> 이렇게 써야 함. 같으면 0 나옴.
* **대입 불가:**
    * 선언할 때 빼고는 `str = "World";` 이렇게 못 넣음. `strcpy(str, "World");` 써야 함.

**[주의/느낀점]**
* 배열 크기 딱 맞게 5로 잡았다가 `\0` 들어갈 자리 없어서 문자 깨짐. 넉넉하게 잡는 게 좋음.
* `strcmp` 같으면 1일 줄 알았는데 0이라서 헷갈림. (차이가 없어서 0이라 생각해야 할 듯!)

## [W13] 포인터 (Pointer)

**[배운 내용]**
* **포인터:** 메모리 주소를 저장하는 변수. (그냥 값이 아니라 '어디에 있는지'를 저장함)
* **연산자:** `&`(주소 구하기), `*`(주소로 가서 값 가져오기, 역참조)
* **Call by Reference:** 함수한테 값 복사해주지 않고 "주소"를 알려줘서 원본을 직접 건드리는 거.
* **포인터와 배열:** 배열 이름(`arr`)은 첫 번째 방의 주소(`&arr[0]`)랑 똑같음.

**[코드/개념 메모]**
* **선언과 사용 구분:**
    * `int *p;` -> 포인터 변수 p를 선언함. (주소 담을 그릇 만듦)
    * `*p = 10;` -> p가 가리키는 곳에 가서 10을 넣음. (실행 코드)
* **주소 연산:**
    * `int a = 10; int *p = &a;`
    * `p`: a의 주소 (ex: 100번지)
    * `*p`: a의 값 (10)
    * `&p`: p라는 변수 자체의 주소
* **Call by Reference 예시:**
    ```c
    void swap(int *a, int *b) {
        int temp = *a; // 주소로 찾아가서 값 꺼냄
        *a = *b;
        *b = temp;
    }
    // 호출할 때 주소(&)를 넘겨야 함: swap(&x, &y);
    ```
    
**[주의/느낀점]**
* `*`가 선언할 때랑 사용할 때랑 의미가 달라서 어려움. 선언할 땐 "포인터야", 사용할 땐 "거기로 가봐".
* 초기화 안 한 포인터(`int *p; *p=10;`) 금지.
* 배열이랑 포인터랑 섞어 쓰니까 너무 어려워서 많은 연습 필요할 듯.

## [W14] 구조체 (Structure)

**[배운 내용]**
* **구조체:** 서로 다른 자료형(int, char, double 등)을 하나로 묶는 것.
* **접근:** 점(`.`) 찍어서 멤버 변수 씀. (`student.age`)
* **화살표 연산자(`->`):** 구조체 "포인터"일 때는 점 대신 화살표 씀. (`p->age`)
* **typedef:** 긴 자료형 이름에 별명 붙여서 짧게 쓰는 것.

**[코드/개념 메모]**
* **기본 문법:**
    ```c
    struct Student {
        char name[20];
        int age;
        double score;
    }; // 세미콜론 필수!
    ```
* **typedef 쓰면 편함:**
    * `typedef struct Student Student;` 해두면 `struct Student s1;` 대신 `Student s1;` 이라고만 써도 됨.
* **점(.) vs 화살표(->):**
    * `st.age = 20;` (일반 변수일 때)
    * `ptr->age = 20;` (포인터 변수일 때) -> `(*ptr).age`랑 같음.

**[주의/느낀점]**
* 배열은 같은 타입만 담는데 구조체는 아니라서 편한 것 같음.
* 구조체 안에 구조체 넣고, 그걸 또 포인터로 가리키니까 코드 길이가 너무 늘어남.
* `->` 연산자 처음엔 어색했는데 쓰다 보니 편한 것 같음.
